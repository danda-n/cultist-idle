# v0.11 Issues — Design Gaps After Full Review

Audit of CONCEPT.md v0.11. Every item here is something an implementer would have to guess,
a decision that hasn't been made, or a contradiction that will cause problems.

Organised by severity. **All items marked BLOCKS must be resolved before the affected system
is implemented.** Items marked GAP need the spec filled in. Items marked EDGE cover
corner cases that must be documented. Items marked POLISH are low-priority wording/math.

---

## ~~BLOCKS IMPLEMENTATION~~ — ALL RESOLVED IN v0.12

### ~~B1 — Devotion collapse vs. global floor-of-3 contradiction~~ ✅ RESOLVED
**Resolution (v0.12):** The two "floors" are different things. §7 updated to clarify:
the 15% offline floor protects the devotion *meter*; the floor-of-3 protects the cultist
*count*. Online: devotion meter can reach 0% (collapse), but at floor-of-3 the cultist is
stunned (§6.4), not lost. See §7.

---

### ~~B2 — Milestone 6 trigger condition ambiguous~~ ✅ RESOLVED
**Resolution (v0.12):** M6 renamed from "The First Research" to "The Branching Path" in §14.
Trigger confirmed: fires when the final Phase 1 node is purchased, immediately unlocking
Phase 2 branches. See §14.

---

### ~~B3 — Voidwreath unlock timing contradictory~~ ✅ RESOLVED
**Resolution (v0.12):** Voidwreath enters the Planet A loot pool at M5 (pool-level gate,
not a per-expedition type). §14 M5 note updated to state this clearly. §11.1 and §12 were
already correct (M5). The confusing "removed" note in §14 has been replaced. See §14 M5 note.

---

### ~~B4 — Hungering Lens dormancy: does it count toward 6-artifact Summoning?~~ ✅ RESOLVED
**Resolution (v0.12):** On expedition return, the ritual circle segment fills immediately —
the Lens is "placed" for all purposes including the M14 check. Segment displays "awaiting
activation" marker until M9. Bonuses activate at M9. §12 Hungering Lens note updated.

---

### ~~B5 — Gateway construction vs. research cost reduction ordering~~ ✅ RESOLVED
**Resolution (v0.12):** -20% cost from "The Opened Way" is non-retroactive (future builds
only). In run 1, most players build at 250A before M5 research. In run 2+, Phase 1
auto-completes so discount applies by default. Clarified in §8.4.

## DESIGN GAPS

### G1 — Harmony bonus threshold never defined
**Affects:** §5, §13 (talent), Trifecta UI

"When all three resources are above a threshold simultaneously, the player earns a Harmony
bonus." The threshold value is never stated. Talent "Harmony bonus threshold lowered by 20%"
lowers it by 20% of what? The mechanic works conceptually but has no number.

**Recommendation:** Add to §5: "Harmony threshold (tunable): base 30% of each resource's
soft cap. All three must simultaneously exceed this for 60+ seconds. Talent lowers threshold
to 24% (20% reduction). Trifecta gauge shows per-resource threshold markers."

---

### G2 — Resource soft cap values never stated
**Affects:** §5, §18, resourceSystem

§5 mentions "soft cap" and §18 references "time-to-cap calculation" in the offline processor.
But no actual cap values are given. Without caps, offline catch-up math cannot be implemented.

**Recommendation:** Add to §5: "Soft caps (tunable): Anima 500, Gnosis 250, Voltis 300.
Above cap, production rate drops to 10% of normal. Caps are per-resource."

---

### G3 — Expedition Choice event pool weights missing
**Affects:** §11.3, expeditionSystem

"Choices are drawn from a weighted pool. Minor choices appear often, major occasionally,
rare choices are memorable surprises." The weight distribution is undefined. The 25-event
pool count (~12 minor / ~8 major / ~5 rare) is listed but probabilities aren't.

**Recommendation:** Add to §11.3: "Draw weights (tunable): Minor 60%, Major 30%, Rare 10%.
Target frequency: ~1 rare choice per 10 expeditions average."

---

### G4 — Cultist passive recruitment rate not promoted from §20
**Affects:** §6.2, cultistSystem

§6.2 says "One new cultist joins every X minutes" with X undefined. §20 Q3 mentions
"Proposed 1 per 20 min base" but never incorporated into the spec.

**Recommendation:** Promote to §6.2: "Passive recruitment: 1 cultist every 20 minutes
(tunable). Active at game start. No unlock required."

---

### G5 — Discipline global action recharge interaction undefined
**Affects:** §7, §13 (talent), devotionSystem

Base: 3 min per-gateway cooldown, 5 min global Discipline cooldown. Talent "+50% faster
recharge" reduces both. But: does using the Global action reset all per-gateway cooldowns
too? With 2 gateways and talent, could a player skip the global cooldown by alternating
per-gateway disciplines?

**Recommendation:** Add to §7: "Global Discipline resets ALL per-gateway discipline
cooldowns simultaneously when used. Per-gateway Discipline does not reset the global timer.
The global action is strictly a convenience upgrade, not an extra charge."

## EDGE CASES

### E1 — Offline devotion floor: per-gateway or global?
**Affects:** §7, §18, offline processor

"Offline devotion floor 15%" — applies to each gateway independently? Or does it prevent
any single gateway from going below 15%? With 2 gateways at different devotion levels,
the floor behavior and the "at risk" UI on return need to be explicit.

**Recommendation:** Clarify §7: "Floor is per-gateway. On return, UI flags all gateways
below 30% devotion as 'At Risk' with red indicator."

---

### E2 — Offline Choice event queue limit undefined
**Affects:** §11.3, §18, offline processor

Multiple expeditions may complete while offline, all returning Choice events. Queue
behavior is unspecified: max size, display order, overflow handling.

**Recommendation:** Add to §18: "Offline Choice queue: max 5 events. On return, displayed
one at a time. If >5 events would queue (rare: requires 5+ expeditions completing offline),
overflow events are resolved as 'Return' (no effect, no choice). Player notified: 'X
expedition results were processed automatically.'"

---

### E3 — Artifact bonuses in run 2+ start timing
**Affects:** §12, §15, prestige/startSystem

"Completed artifacts survive Rehearsal." But when do their bonuses activate in a new run?
Cindermark (+30% Anima) persisting into run 2 — does it apply from game start or only
after reaching the original unlock milestone?

**Recommendation:** Add to §15: "Artifact bonuses from prior runs activate at game start,
not when the artifact was originally unlocked. Run 2 begins with all carried-over artifact
bonuses active immediately."

---

### E4 — Expedition keystone talent timing
**Affects:** §11.1, §13 (talent keystone), expeditionSystem

"Base cap: 2 parallel expeditions. Talent keystone upgrades to 3." Keystones require Boons
(earned at Rehearsal). Expeditions start at M4, first Rehearsal is M8. So run 1 is always
capped at 2. This is presumably intentional but never stated.

**Recommendation:** Add to §11.1: "3-expedition cap requires the Expedition talent keystone
(costs 2 Boons). First available after M8 Rehearsal. Run 1 is always capped at 2 parallel
expeditions."

---

### E5 — Corruption debuff at resource production floor
**Affects:** §11.4, resourceSystem

"No corruption can reduce a resource below its natural production floor." But what IS the
floor? If Anima production is, say, 0.1/min from base altar output, and Ash-Taint debuff
is -0.5/min, does Anima production go to 0 (floored) or 0.1 - 0.5 = negative (capped)?
The floor rule prevents negative rates, but the UI should still show the debuff as active.

**Recommendation:** Add to §11.4: "Corruption debuffs always display as active in the UI
even when floored. Production rates floor at 0/min (never negative). The debuff's full
cost still counts for cleansing calculations."

## POLISH / MATH

### P1 — Blood Accelerant: one bonus or two?
**Affects:** §10 (Phase 2 Acceleration branch)

"Blood Accelerant — Expedition timers -20%" appears alongside "Voltis yield per channel
+30%?" in some reads of the document. If both bonuses are from one 80G node, it's
overloaded vs. other nodes. Confirm: Blood Accelerant is expedition timers only; Voltis
yield is The Hungry Gate (node 2).

---

### P2 — Harmony multiplier base value and talent interaction
**Affects:** §5, §13

§5: Harmony is "+15-25% production multiplier." §13 talent: "Harmony bonus multiplier +10%
additional." Is base 15-25% (range), and talent adds 10pp? Or is base 20% flat?

**Recommendation:** Specify: "Base Harmony bonus: +20% all production (tunable). Trifecta
Resonance talent: +10pp additional = +30% total. Unbinding artifact: doubled (= +40%,
or +60% with talent)."

---

### P3 — Cindermark "yields doubled" wording conflict
**Affects:** §6.5, §12

§6.5: "Cindermark doubles total sacrifice production retroactively." §12 table: "cultist
sacrifice yields doubled." The §12 wording implies future sacrifices only; §6.5 is explicit
about retroactive. Unify the language.

**Recommendation:** Update §12 Cindermark effect: "All sacrifice Anima/min doubled
retroactively — applies to all active sacrifices, not just future ones."

---

### P4 — Gnosis budget math: source of "~500 earnable" not documented
**Affects:** §10 (Gnosis budget verification)

The budget check passes (480G needed, ~500G earnable, ~20G surplus) but the "~500 earnable"
figure has no derivation. When production rates are tuned in `src/data/`, this needs to be
re-verified against actual numbers.

**Note:** Marked ✓ in the doc, but is an assumption not a derivation. Flag for re-check
when channelSystem.ts numbers are confirmed.

---

### P5 — Devotion decay per-tick conversion not specified
**Affects:** §7, devotionSystem

Table shows rates as %/min (e.g., 0.5%/min normal). tick() runs every 100ms.
Implementer must convert: 0.5%/min = 0.5/60000 per ms. Should be explicit to prevent
rounding errors compounding across long offline periods.

**Recommendation:** Add note to §7 or §18: "All rates stored as per-minute for readability.
Tick conversion: multiply by (deltaMs / 60000). Example: 0.5%/min decay applied as
state.devotion -= 0.5 × (deltaMs / 60000)."

---

### P6 — Trifecta UI shape never specified
**Affects:** §5, §16

"Three-bar gauge (red = needs attention)" vs. "triangular gauge" — shape is inconsistent
across sections. Minor: pick one.

---

### P7 — Multi-resource artifact progress bar bottleneck rule
**Affects:** §12

"Cost progress bar showing current resource % toward completion." Unclear for artifacts with
3 resource costs (Anima + Gnosis + Voltis). Which resource's % shows?

**Recommendation:** "Show the lowest-% resource (bottleneck). Tooltip shows all three
individually."
