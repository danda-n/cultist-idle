# v0.11 Issues — Design Gaps After Full Review

Audit of CONCEPT.md v0.11. Every item here is something an implementer would have to guess,
a decision that hasn't been made, or a contradiction that will cause problems.

Organised by severity. **All items marked BLOCKS must be resolved before the affected system
is implemented.** Items marked GAP need the spec filled in. Items marked EDGE cover
corner cases that must be documented. Items marked POLISH are low-priority wording/math.

---

## ~~BLOCKS IMPLEMENTATION~~ — ALL RESOLVED IN v0.12

### ~~B1 — Devotion collapse vs. global floor-of-3 contradiction~~ ✅ RESOLVED
**Resolution (v0.12):** The two "floors" are different things. §7 updated to clarify:
the 15% offline floor protects the devotion *meter*; the floor-of-3 protects the cultist
*count*. Online: devotion meter can reach 0% (collapse), but at floor-of-3 the cultist is
stunned (§6.4), not lost. See §7.

---

### ~~B2 — Milestone 6 trigger condition ambiguous~~ ✅ RESOLVED
**Resolution (v0.12):** M6 renamed from "The First Research" to "The Branching Path" in §14.
Trigger confirmed: fires when the final Phase 1 node is purchased, immediately unlocking
Phase 2 branches. See §14.

---

### ~~B3 — Voidwreath unlock timing contradictory~~ ✅ RESOLVED
**Resolution (v0.12):** Voidwreath enters the Planet A loot pool at M5 (pool-level gate,
not a per-expedition type). §14 M5 note updated to state this clearly. §11.1 and §12 were
already correct (M5). The confusing "removed" note in §14 has been replaced. See §14 M5 note.

---

### ~~B4 — Hungering Lens dormancy: does it count toward 6-artifact Summoning?~~ ✅ RESOLVED
**Resolution (v0.12):** On expedition return, the ritual circle segment fills immediately —
the Lens is "placed" for all purposes including the M14 check. Segment displays "awaiting
activation" marker until M9. Bonuses activate at M9. §12 Hungering Lens note updated.

---

### ~~B5 — Gateway construction vs. research cost reduction ordering~~ ✅ RESOLVED
**Resolution (v0.12):** -20% cost from "The Opened Way" is non-retroactive (future builds
only). In run 1, most players build at 250A before M5 research. In run 2+, Phase 1
auto-completes so discount applies by default. Clarified in §8.4.

## ~~DESIGN GAPS~~ — ALL RESOLVED IN v0.13

### ~~G1 — Harmony bonus threshold never defined~~ ✅ RESOLVED
**Resolution (v0.13):** Base threshold = 30% of each resource's soft cap (Anima 150,
Gnosis 75, Voltis 90). Talent reduces to 24% of soft cap. All three must hold above
threshold simultaneously for 60+ seconds. Added to §5.

---

### ~~G2 — Resource soft cap values never stated~~ ✅ RESOLVED
**Resolution (v0.13):** Anima 500, Gnosis 250, Voltis 300. Tunable, live in `src/data/`.
Added to §5.

---

### ~~G3 — Expedition Choice event pool weights missing~~ ✅ RESOLVED
**Resolution (v0.13):** Minor 60%, Major 30%, Rare 10%. Target ~1 rare per 10 returns.
Added to §11.3.

---

### ~~G4 — Cultist passive recruitment rate not promoted from §20~~ ✅ RESOLVED
**Resolution (v0.13):** 1 cultist every 20 minutes, tunable, active from game start.
"Can be improved via research tree" removed — no research node covers this. Milestone
bonuses are the primary recruitment mechanic. Updated in §6.2.

---

### ~~G5 — Discipline global action recharge interaction undefined~~ ✅ RESOLVED
**Resolution (v0.13):** Global Discipline resets all per-gateway cooldowns when used.
Per-gateway Discipline does not reset the global timer. Documented in §7.

---

## ~~EDGE CASES~~ — ALL RESOLVED IN v0.14

### ~~E1 — Offline devotion floor: per-gateway or global?~~ ✅ RESOLVED
**Resolution (v0.14):** Per-gateway (established in B1). On return, gateways below 30%
devotion flagged with red indicator. 30% = ~30 min before collapse at normal rate. Added
to §7.

---

### ~~E2 — Offline Choice event queue limit undefined~~ ✅ RESOLVED
**Resolution (v0.14):** Max 5 queued events, displayed one at a time on return. Overflow
auto-resolves as "Return" (safe, no loot). Player notified of count. Added to §18.

---

### ~~E3 — Artifact bonuses in run 2+ start timing~~ ✅ RESOLVED
**Resolution (v0.14):** All persisted artifact bonuses activate at game start — no gate,
no delay. Exception: Hungering Lens still requires Planet B open. Added to §15.

---

### ~~E4 — Expedition keystone: run 1 cap never stated~~ ✅ RESOLVED
**Resolution (v0.14):** Run 1 cap is always 2. 3rd slot requires Expedition keystone
(2 Boons), first available after M8 Rehearsal. Stated explicitly in §11.1.

---

### ~~E5 — Corruption debuff at resource production floor~~ ✅ RESOLVED
**Resolution (v0.14):** Natural floor = 0/min. Debuff always shows active in UI even when
floored. Full 200 Gnosis cleansing cost always applies. Added to §11.4.

---

## ~~POLISH / MATH~~ — ALL RESOLVED IN v0.15

### ~~P1 — Blood Accelerant: one bonus or two?~~ ✅ RESOLVED
**Resolution (v0.15):** Blood Accelerant = expedition timers -20% only (80G). Voltis yield
+30% is The Hungry Gate node 2 (100G). Clarified in §10 table with "(Voltis-only)" note.

---

### ~~P2 — Harmony multiplier base value unclear~~ ✅ RESOLVED
**Resolution (v0.15):** Base +20% all production. Trifecta Resonance +10pp = +30% total.
Unbinding doubles active bonus (+40% base, +60% with talent). Specified in §5.

---

### ~~P3 — Cindermark wording conflict §6.5 vs §12~~ ✅ RESOLVED
**Resolution (v0.15):** Both §6.5 and §12 now read "all active sacrifice yields doubled
retroactively." Example added to §6.5 for clarity.

---

### ~~P4 — Gnosis budget "~500 earnable" undocumented~~ ✅ RESOLVED
**Resolution (v0.15):** Derivation note added to §10 budget section. Flagged for re-check
against final `src/data/` channel rates during balance pass.

---

### ~~P5 — Devotion decay per-tick conversion missing~~ ✅ RESOLVED
**Resolution (v0.15):** Tick formula added to §7: `devotion -= rate × (deltaMs / 60000)`.
Modifiers apply multiplicatively to the per-minute rate before conversion.

---

### ~~P6 — Trifecta UI shape inconsistent~~ ✅ RESOLVED
**Resolution (v0.15):** Canonical shape is **triangular gauge** — three segments, one per
resource, GREEN/YELLOW/RED per threshold. Specified in §5.

---

### ~~P7 — Multi-resource artifact progress bar rule~~ ✅ RESOLVED
**Resolution (v0.15):** Bar shows bottleneck resource (lowest % of requirement). Tooltip
breaks down all three individually. Specified in §12.
