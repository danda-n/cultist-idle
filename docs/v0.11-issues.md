# v0.11 Issues — Design Gaps After Full Review

Audit of CONCEPT.md v0.11. Every item here is something an implementer would have to guess,
a decision that hasn't been made, or a contradiction that will cause problems.

Organised by severity. **All items marked BLOCKS must be resolved before the affected system
is implemented.** Items marked GAP need the spec filled in. Items marked EDGE cover
corner cases that must be documented. Items marked POLISH are low-priority wording/math.

---

## ~~BLOCKS IMPLEMENTATION~~ — ALL RESOLVED IN v0.12

### ~~B1 — Devotion collapse vs. global floor-of-3 contradiction~~ ✅ RESOLVED
**Resolution (v0.12):** The two "floors" are different things. §7 updated to clarify:
the 15% offline floor protects the devotion *meter*; the floor-of-3 protects the cultist
*count*. Online: devotion meter can reach 0% (collapse), but at floor-of-3 the cultist is
stunned (§6.4), not lost. See §7.

---

### ~~B2 — Milestone 6 trigger condition ambiguous~~ ✅ RESOLVED
**Resolution (v0.12):** M6 renamed from "The First Research" to "The Branching Path" in §14.
Trigger confirmed: fires when the final Phase 1 node is purchased, immediately unlocking
Phase 2 branches. See §14.

---

### ~~B3 — Voidwreath unlock timing contradictory~~ ✅ RESOLVED
**Resolution (v0.12):** Voidwreath enters the Planet A loot pool at M5 (pool-level gate,
not a per-expedition type). §14 M5 note updated to state this clearly. §11.1 and §12 were
already correct (M5). The confusing "removed" note in §14 has been replaced. See §14 M5 note.

---

### ~~B4 — Hungering Lens dormancy: does it count toward 6-artifact Summoning?~~ ✅ RESOLVED
**Resolution (v0.12):** On expedition return, the ritual circle segment fills immediately —
the Lens is "placed" for all purposes including the M14 check. Segment displays "awaiting
activation" marker until M9. Bonuses activate at M9. §12 Hungering Lens note updated.

---

### ~~B5 — Gateway construction vs. research cost reduction ordering~~ ✅ RESOLVED
**Resolution (v0.12):** -20% cost from "The Opened Way" is non-retroactive (future builds
only). In run 1, most players build at 250A before M5 research. In run 2+, Phase 1
auto-completes so discount applies by default. Clarified in §8.4.

Added to §11.3.

## POLISH / MATH

### P1 — Blood Accelerant: one bonus or two?
**Affects:** §10 (Phase 2 Acceleration branch)

"Blood Accelerant — Expedition timers -20%" appears alongside "Voltis yield per channel
+30%?" in some reads of the document. If both bonuses are from one 80G node, it's
overloaded vs. other nodes. Confirm: Blood Accelerant is expedition timers only; Voltis
yield is The Hungry Gate (node 2).

---

### P2 — Harmony multiplier base value and talent interaction
**Affects:** §5, §13

§5: Harmony is "+15-25% production multiplier." §13 talent: "Harmony bonus multiplier +10%
additional." Is base 15-25% (range), and talent adds 10pp? Or is base 20% flat?

**Recommendation:** Specify: "Base Harmony bonus: +20% all production (tunable). Trifecta
Resonance talent: +10pp additional = +30% total. Unbinding artifact: doubled (= +40%,
or +60% with talent)."

---

### P3 — Cindermark "yields doubled" wording conflict
**Affects:** §6.5, §12

§6.5: "Cindermark doubles total sacrifice production retroactively." §12 table: "cultist
sacrifice yields doubled." The §12 wording implies future sacrifices only; §6.5 is explicit
about retroactive. Unify the language.

**Recommendation:** Update §12 Cindermark effect: "All sacrifice Anima/min doubled
retroactively — applies to all active sacrifices, not just future ones."

---

### P4 — Gnosis budget math: source of "~500 earnable" not documented
**Affects:** §10 (Gnosis budget verification)

The budget check passes (480G needed, ~500G earnable, ~20G surplus) but the "~500 earnable"
figure has no derivation. When production rates are tuned in `src/data/`, this needs to be
re-verified against actual numbers.

**Note:** Marked ✓ in the doc, but is an assumption not a derivation. Flag for re-check
when channelSystem.ts numbers are confirmed.

---

### P5 — Devotion decay per-tick conversion not specified
**Affects:** §7, devotionSystem

Table shows rates as %/min (e.g., 0.5%/min normal). tick() runs every 100ms.
Implementer must convert: 0.5%/min = 0.5/60000 per ms. Should be explicit to prevent
rounding errors compounding across long offline periods.

**Recommendation:** Add note to §7 or §18: "All rates stored as per-minute for readability.
Tick conversion: multiply by (deltaMs / 60000). Example: 0.5%/min decay applied as
state.devotion -= 0.5 × (deltaMs / 60000)."

---

### P6 — Trifecta UI shape never specified
**Affects:** §5, §16

"Three-bar gauge (red = needs attention)" vs. "triangular gauge" — shape is inconsistent
across sections. Minor: pick one.

---

### P7 — Multi-resource artifact progress bar bottleneck rule
**Affects:** §12

"Cost progress bar showing current resource % toward completion." Unclear for artifacts with
3 resource costs (Anima + Gnosis + Voltis). Which resource's % shows?

**Recommendation:** "Show the lowest-% resource (bottleneck). Tooltip shows all three
individually."
